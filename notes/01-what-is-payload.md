# Payload CMS 是什么

> 整理自官方文档 + 社区资料，记录于学习过程中。

## 一句话定义

**Payload 是一个基于 Next.js 的开源全栈框架**，从 Headless CMS 起家，现已演化为完整的应用开发框架。

- 技术栈：Node.js + TypeScript + React + Next.js
- 开源协议：MIT
- 托管方式：自托管（可部署到 Vercel 等 Serverless 平台）
- 重要动态：**已被 Figma 收购**（2025年）

---

## 在软件架构中的位置

```
┌────────────────────────────────────┐
│         前端 / 客户端               │
│  (Next.js, React, iOS, Android…)   │
└───────────────┬────────────────────┘
                │ REST / GraphQL API
┌───────────────▼────────────────────┐  ← Payload 在这一层
│           Payload CMS              │
│  ┌─────────────┐  ┌─────────────┐  │
│  │  Admin UI   │  │  API 层     │  │
│  │ (React自动  │  │ REST/GraphQL│  │
│  │  生成)      │  │ /Node.js    │  │
│  └─────────────┘  └─────────────┘  │
│  ┌────────────────────────────────┐ │
│  │  业务逻辑层（Hooks / Plugins） │ │
│  └────────────────────────────────┘ │
└───────────────┬────────────────────┘
                │
┌───────────────▼────────────────────┐
│           数据库                    │
│     (PostgreSQL / MongoDB)         │
└────────────────────────────────────┘
```

Payload 处于 **BFF / 后端应用层**，是连接数据库和前端的中间层，负责：
- 数据建模（Collections / Globals）
- API 暴露（REST / GraphQL / Node.js 直连）
- 权限控制（字段级）
- 内容管理（Admin UI）

## 核心能力

| 能力 | 说明 |
|---|---|
| **自动生成 Admin UI** | 写一份 Config，立刻得到完整的后台管理界面（React 组件，可深度定制） |
| **自动生成 API** | 同一份 Config，自动生成 REST、GraphQL、Node.js 直连 DB 三套 API |
| **数据库管理** | 自动建表/迁移、事务、索引，支持 PostgreSQL 和 MongoDB |
| **认证系统** | 内置完整的用户认证，可直接复用在自己的业务中 |
| **访问控制** | 细粒度权限系统，字段级别的读写控制 |
| **文件/图片管理** | 上传、裁剪、焦点选择、媒体库 |
| **实时预览** | 编辑内容时前端同步实时渲染 |
| **Code-first** | 所有配置都是代码，纳入 Git 版本控制，无"点点点配置" |

## 四大典型使用场景

1. **Headless CMS**：给 Next.js 网站提供内容管理后台，替代 Contentful/Sanity 等 SaaS
2. **企业内部工具**：快速搭建有权限控制 + Admin UI 的内部管理系统
3. **Headless 电商**：配合 Stripe 等支付处理，商品/内容/媒体统一管理
4. **数字资产管理（DAM）**：大批量文件管理、版本控制、标签检索

## 与其他产品的关键区别

| 对比维度 | WordPress | Contentful/Sanity（SaaS） | Payload |
|---|---|---|---|
| 架构 | 传统耦合型 | Headless，但闭源托管 | Headless，开源自托管 |
| 数据归属 | 自己 | 在厂商服务器 | 完全自己 |
| 定制度 | 插件堆叠 | 有限 | 代码级完全可控 |
| 适用人群 | 非技术用户 | 开发者+编辑 | 以开发者为主 |
| 费用 | 免费+插件费 | 按用量计费（贵） | MIT 开源，自己出服务器 |

## 适合 / 不适合的情况

**适合：**
- 在用 Next.js，需要一个 CMS，不想为 SaaS 付月费
- 重视数据隐私/所有权，不愿数据放在第三方
- 需要在标准 CMS 之外写自定义业务逻辑
- 快速搭建带后台管理界面的企业内部工具
- 想部署到 Vercel 等 Serverless 平台

**不适合：**
- 项目简单到 Webflow/Framer 就够用
- 纯前端静态站点，不需要 Admin UI
- 已有完整数据库，只需要可视化数据
