Slug × AI Task 架构原理

两种 Slug 的含义 · 稳定调用链 · Agent 任务执行全流程

① Collection Slug（代码层）

  • • 在 CollectionConfig 里定义 slug: 'tags'
  • • 自动生成 REST API：/api/tags
  • • Admin 菜单 URL：/admin/collections/tags
  • • 代码里用 payload.find({collection:'tags'})
  • • 用户永远看不到这个 slug，它在代码里固定

② Document Slug Field（数据层）

  • • 你在 Admin UI 里为每个文档手动填的字段
  • • 例：Tags 文档 name="电子签" → slug="electronic-sign"
  • • 例：AgentTask name="生成摘要" → slug="generate-excerpt"
  • • 数据库 ID 可能因迁移变化，slug 你自己控制
  • • 建议填写规则：小写英文+连字符，简洁明确

③ AI Task 调用（运行时）

  • • POST /api/agent-tasks/generate-excerpt/run
  • • Endpoint 先尝试 findByID，ID 不存在则 findBySlug
  • • 支持传 inputs:{url:"..."} 填充 prompt 变量
  • • 立即返回 jobId，Job 后台异步执行
  • • 结果通过 agent-task-runs 记录查询，不阻塞调用方
