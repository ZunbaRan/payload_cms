Payload Dev 模式 Schema 同步流程

为什么会出现 "Is tags table created or renamed?" 的交互提示

普通步骤 / Collection
关键节点 / 新增内容
数据库辅助表（自动生成）
relationship 外键连接

为什么出现交互提示？
Payload 检测到旧库有 images_texts 和 keywords_texts，新 Schema 里出现了 tags 表——它无法自动判断是"新建"还是"从旧表改名"（两者名字差太多）。选 create table 后，旧辅助表会被 DROP，新 tags 表被 CREATE，关联表 images_rels / keywords_rels 被建立，全部完成。
