Payload CMS 插件化数字员工系统架构

基于 MCP 协议的 AI Agent 与业务 Plugin 集成方案 · 同事架构思路整理

MCP 接入层 — 标准化

  • • 每个 Plugin 通过 @payloadcms/plugin-mcp 暴露接口
  • • Agent 统一用 MCP 协议调用，接口一致
  • • 无需对接各家 SaaS 的私有 API
  • • API Key 绑定权限，精确控制 Agent 能力范围

Plugin 开发模型 — 可插拔

  • • 一个 Plugin = Collections + Tools + Skills
  • • npm install 安装，plugins[] 注册即生效
  • • 安装后 Admin UI / API 自动出现
  • • 业务逻辑封装成"比较重的 Tool"（审批流、状态机）

统一平台价值 — 一个实例

  • • 一个 Payload 实例承载多个业务模块
  • • 统一权限 / 认证 / 数据库连接
  • • Collection 间可跨业务关联数据
  • • 数据完全自主，不被 SaaS 厂商锁定
