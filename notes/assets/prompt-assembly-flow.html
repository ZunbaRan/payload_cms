Prompt 拼接全流程 — {{变量}} 如何注入业务字段

Admin UI 配置 → 运行时变量合并 → renderTemplate → AI 接收最终 prompt

变量声明 vs 传入

  • • Admin UI 里的 variables[] 是声明：告诉调用方"我需要哪些变量"
  • • 每条变量有 key + defaultValue
  • • 调用 /run 时通过 inputs:{} 覆盖默认值
  • • Object.assign(mergedVars, runInputs) 运行时值优先
  • • 没传的变量用 defaultValue，都没有则空字符串

{{变量}} 替换规则

  • • 正则：/\{\{\s*([\w.-]+)\s*\}\}/g
  • • {{title}} → mergedVars["title"] 的值
  • • 未声明的变量 → 保留原文 {{xxx}} 不替换
  • • 值为 null/undefined → 替换为空字符串 ""
  • • 支持嵌套词：{{article.title}} 也可以

两种 Task 的差异

  • • processAgentTaskRun：变量全靠调用方 inputs 传入
  • • processTaskRun：变量从素材库自动抓取（title/keywords/category）
  • • AgentTask 更灵活：agent 自主调用 skill/bash
  • • ContentTask 更固定：固定 pipeline 生成文章存库
  • • 两者最终都调 ai.generate({systemPrompt, userPrompt})
